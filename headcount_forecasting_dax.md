## Data Model Setup

Import the four CSV files generated by `generate_headcount_data.py`.  
Recommended table names in Power BI Desktop:

| File | Table Name |
|---|---|
| headcount_monthly.csv | Headcount_Monthly |
| employees.csv | Employees |
| separations.csv | Separations |
| new_hires.csv | New_Hires |

**Date Table** — Create a dedicated date table using the DAX below, then relate it
to `Headcount_Monthly[month_date]` (many-to-one).

```dax
Date_Table =
ADDCOLUMNS(
    CALENDAR(DATE(2022, 1, 1), DATE(2024, 12, 31)),
    "Year",        YEAR([Date]),
    "Month_Num",   MONTH([Date]),
    "Month_Name",  FORMAT([Date], "MMM"),
    "Month_Year",  FORMAT([Date], "MMM YYYY"),
    "Quarter",     "Q" & QUARTER([Date]),
    "Year_Quarter","Q" & QUARTER([Date]) & " " & YEAR([Date])
)
```

**Relationships:**
- `Date_Table[Date]` → `Headcount_Monthly[month_date]` (Many-to-One, active)
- `Headcount_Monthly[department]` → `Employees[department]` (Many-to-One, inactive — use USERELATIONSHIP if needed)

## Section 1: Core Headcount Measures

### 1.1 Closing Headcount
Total active headcount at the end of the selected period.

```dax
Closing Headcount =
SUM(Headcount_Monthly[closing_headcount])
```

---

### 1.2 Opening Headcount
Headcount at the start of the current period (prior month's closing headcount).
Uses DATEADD to shift back one month in the date context.

```dax
Opening Headcount =
CALCULATE(
    SUM(Headcount_Monthly[closing_headcount]),
    DATEADD(Date_Table[Date], -1, MONTH)
)
```

---

### 1.3 Total New Hires

```dax
Total New Hires =
SUM(Headcount_Monthly[new_hires])
```

---

### 1.4 Total Separations

```dax
Total Separations =
SUM(Headcount_Monthly[separations])
```

---

### 1.5 Net Headcount Change
The change in headcount for the selected period.

```dax
Net Headcount Change =
[Closing Headcount] - [Opening Headcount]
```

---

### 1.6 Net Headcount Change % (MoM)

```dax
Net Headcount Change % =
DIVIDE(
    [Net Headcount Change],
    [Opening Headcount],
    0
)
```
*Format as Percentage in the model.*

## Section 2: Attrition Measures

### 2.1 Attrition Rate % (Annualized)
Annualized attrition based on separations relative to average headcount in the period.
Grounded in SHRM 2023 benchmark methodology for financial services (17.1% avg voluntary).

```dax
Attrition Rate % =
VAR TotalSeps   = [Total Separations]
VAR AvgHeadcount =
    AVERAGEX(
        VALUES(Headcount_Monthly[month_date]),
        CALCULATE(SUM(Headcount_Monthly[closing_headcount]))
    )
VAR MonthsInPeriod =
    DISTINCTCOUNT(Headcount_Monthly[month_date])
RETURN
    DIVIDE(
        TotalSeps * 12,
        AvgHeadcount * MonthsInPeriod,
        0
    )
```
*Format as Percentage (1 decimal place).*

---

### 2.2 Rolling 3-Month Attrition Rate
Smooths monthly volatility to reveal underlying trends.

```dax
Rolling 3M Attrition Rate % =
VAR SelectedDate = MAX(Date_Table[Date])
VAR ThreeMonthsAgo = EDATE(SelectedDate, -2)
VAR Seps =
    CALCULATE(
        SUM(Headcount_Monthly[separations]),
        Date_Table[Date] >= ThreeMonthsAgo,
        Date_Table[Date] <= SelectedDate
    )
VAR AvgHC =
    CALCULATE(
        AVERAGEX(
            VALUES(Headcount_Monthly[month_date]),
            CALCULATE(SUM(Headcount_Monthly[closing_headcount]))
        ),
        Date_Table[Date] >= ThreeMonthsAgo,
        Date_Table[Date] <= SelectedDate
    )
RETURN
    DIVIDE(Seps * 4, AvgHC, 0)
```
*Format as Percentage.*

---

### 2.3 Voluntary Separations

```dax
Voluntary Separations =
CALCULATE(
    COUNTROWS(Separations),
    Separations[separation_type] = "Voluntary"
)
```

---

### 2.4 Involuntary Separations

```dax
Involuntary Separations =
CALCULATE(
    COUNTROWS(Separations),
    Separations[separation_type] = "Involuntary"
)
```

---

### 2.5 Voluntary Attrition Rate %

```dax
Voluntary Attrition Rate % =
VAR AvgHC =
    AVERAGEX(
        VALUES(Headcount_Monthly[month_date]),
        CALCULATE(SUM(Headcount_Monthly[closing_headcount]))
    )
VAR MonthsInPeriod = DISTINCTCOUNT(Headcount_Monthly[month_date])
RETURN
    DIVIDE(
        [Voluntary Separations] * 12,
        AvgHC * MonthsInPeriod,
        0
    )
```

## Section 3: Year-over-Year Measures

### 3.1 Headcount — Same Period Last Year

```dax
Headcount SPLY =
CALCULATE(
    [Closing Headcount],
    SAMEPERIODLASTYEAR(Date_Table[Date])
)
```

---

### 3.2 YoY Headcount Change

```dax
YoY Headcount Change =
[Closing Headcount] - [Headcount SPLY]
```

---

### 3.3 YoY Headcount Change %

```dax
YoY Headcount Change % =
DIVIDE(
    [YoY Headcount Change],
    [Headcount SPLY],
    0
)
```
*Format as Percentage.*

## Section 4: Hiring Efficiency Measures

### 4.1 Hire-to-Separation Ratio
Values > 1 indicate net headcount growth; < 1 indicates net decline.

```dax
Hire to Separation Ratio =
DIVIDE(
    [Total New Hires],
    [Total Separations],
    0
)
```

---

### 4.2 Headcount Stability Rate %
Proportion of workforce that remained active through the selected period.
Inverse of attrition; useful for board-level reporting.

```dax
Headcount Stability Rate % =
1 - [Attrition Rate %]
```

## Section 5: Headcount Forecasting Measures

### 5.1 Avg Monthly Net Headcount Change (Trailing 6 Months)
Calculates the average monthly net change used as the basis for forward projections.

```dax
Avg Monthly Net Change (6M) =
VAR LatestDate = MAX(Date_Table[Date])
VAR SixMonthsAgo = EDATE(LatestDate, -5)
VAR HC_End =
    CALCULATE(
        SUM(Headcount_Monthly[closing_headcount]),
        Date_Table[Date] = LatestDate
    )
VAR HC_Start =
    CALCULATE(
        SUM(Headcount_Monthly[closing_headcount]),
        Date_Table[Date] = SixMonthsAgo
    )
RETURN
    DIVIDE(HC_End - HC_Start, 6, 0)
```

---

### 5.2 Forecast: Month +1

```dax
Forecast Month +1 =
VAR LastHC = CALCULATE(
    SUM(Headcount_Monthly[closing_headcount]),
    Date_Table[Date] = MAX(Date_Table[Date])
)
RETURN
    ROUND(LastHC + [Avg Monthly Net Change (6M)], 0)
```

---

### 5.3 Forecast: Month +2

```dax
Forecast Month +2 =
VAR LastHC = CALCULATE(
    SUM(Headcount_Monthly[closing_headcount]),
    Date_Table[Date] = MAX(Date_Table[Date])
)
RETURN
    ROUND(LastHC + ([Avg Monthly Net Change (6M)] * 2), 0)
```

---

### 5.4 Forecast: Month +3

```dax
Forecast Month +3 =
VAR LastHC = CALCULATE(
    SUM(Headcount_Monthly[closing_headcount]),
    Date_Table[Date] = MAX(Date_Table[Date])
)
RETURN
    ROUND(LastHC + ([Avg Monthly Net Change (6M)] * 3), 0)
```

---

### 5.5 Forecast vs Actual Variance
Use this measure after the forecast period has passed to evaluate accuracy.

```dax
Forecast Variance =
[Closing Headcount] - [Forecast Month +1]
```

## Section 6: Supporting Card/KPI Measures

### 6.1 Total Active Employees (Employees Table)
Point-in-time active count from the employee master (does not respect date slicer).

```dax
Total Active Employees =
CALCULATE(
    COUNTROWS(Employees),
    Employees[status] = "Active"
)
```

---

### 6.2 Headcount Label (for chart annotations)

```dax
Headcount Label =
FORMAT([Closing Headcount], "#,0") & " employees"
```

---

### 6.3 Attrition Benchmark Flag
Returns "Above Benchmark" or "Within Benchmark" relative to SHRM 2023 financial
services average of 17.1% voluntary attrition.

```dax
Attrition Benchmark Flag =
IF(
    [Voluntary Attrition Rate %] > 0.171,
    "Above Benchmark",
    "Within Benchmark"
)
```

## Notes

- All measures use the active relationship between `Date_Table[Date]` and
  `Headcount_Monthly[month_date]`. Slicers on the report pages drive the date context.
- For department-level forecasting, wrap forecast measures in
  `CALCULATE(..., ALLEXCEPT(Headcount_Monthly, Headcount_Monthly[department]))`.
- Attrition rate measures annualize by multiplying by 12/months-in-period to allow
  fair comparison across single-month and full-year selections.
